Modern CSS en kleurruimtes

Moderne CSS-kleurruimtes zoals OKLCH geven webontwikkelaars veel meer controle over gradients, kleurintensiteit en hoe kleuren daadwerkelijk worden waargenomen op moderne schermen.

Wat we kunnen leren van Sanne's presentatie over gradients en color spaces

Als webontwikkelaars zijn we gewend om kleuren te definiëren in hex, rgb() of hsl().

We vullen een verloopje in met linear-gradient en vertrouwen erop dat de browser het mooi oplost.

Maar tijdens de presentatie "WDD - kleurtjes" liet Sanne zien dat de manier waarop browsers traditioneel met kleur omgaan flink achterloopt op moderne schermen.

Het web heeft er krachtige nieuwe kleurruimtes en tools bij gekregen om kleuren écht te laten spatten.

De beperking van de traditionele RGB-gradient

Wanneer je een lineaire gradient maakt tussen twee felle kleuren in de standaard RGB-kleurruimte zoals van geel naar paars, gebeurt er iets geks in het midden.

Er ontstaat vaak een doffe, grijsachtige overgang.

Hoe komt dat?

  • Een RGB-gradient volgt een rechte lijn door het grijze midden van de kleurkubus.
  • De browser berekent het gemiddelde van de rode, groene en blauwe kanalen, waardoor kleuren onderweg helderheid en verzadiging verliezen.

Dit wordt ook wel de “dead zone” van een gradient genoemd.

HSL: rekenen in een cirkel

Een alternatief dat al langer bestaat is HSL (Hue, Saturation, Lightness).

Hoewel HSL voor mensen logischer voelt om mee te werken, lost het het gradient-probleem niet perfect op.

  • Een HSL-gradient volgt een cirkel met dezelfde verzadiging en lichtheid.
  • Daardoor beweegt de berekening door het volledige kleurenspectrum in plaats van een natuurlijke overgang te maken.

Dit resulteert vaak in onverwachte tussenkleuren in je verloop.

De oplossing: oklch en de longer hue syntax

Met moderne CSS-kleurruimtes zoals oklch hebben we nu gradients die gebaseerd zijn op hoe het menselijk oog kleur daadwerkelijk waarneemt.

Bovendien geeft CSS ons nu controle over hoe er door de kleurencirkel wordt gereisd.

In de presentatie zagen we deze code: 

div:nth-of-type(4) {
  background-image: linear-gradient(
    90deg in oklch longer hue,
    #fc0,
    #f0c
  );
}

Wat betekent dit?

  • in oklch zorgt ervoor dat de browser tussenliggende kleuren berekent binnen de OKLCH-kleurruimte.
  • Hierdoor blijven gradients helder, levendig en zonder grijze dead zone.
  • longer hue vertelt de browser om de lange route door de kleurencirkel te nemen.

Zo kun je met slechts twee kleuren een volledige regenbooggradient maken.

Checken of de browser en hardware het aankunnen

Niet elk scherm kan moderne kleurruimtes zoals display-p3 weergeven.

Daarnaast ondersteunen oudere browsers niet altijd de nieuwste CSS-syntax.

Gelukkig kunnen we dit tegenwoordig volledig native oplossen met CSS.

Checken of het device het gamut ondersteunt

:root {
  @media (color-gamut: p3) {
    --green: color(display-p3 0 1 0);
  }
}

Hiermee controleer je of het scherm rijkere kleuren kan weergeven.

Checken of de browser de syntax begrijpt

:root {
  @supports (color(display-p3 0 1 0)) {
    --green: color(display-p3 0 1 0);
  }
}

Met @supports controleer je of de browser de nieuwe CSS-syntax begrijpt.

Conclusie

Moderne CSS-kleurruimtes geven developers veel meer controle over hoe gradients en kleuren eruitzien.

  • Gradients worden helderder en natuurlijker
  • Je voorkomt modderige kleurtransities
  • Met minimale code maak je complexe kleurverlopen
  • Feature detection werkt volledig native in CSS

Kleur op het web is daardoor niet langer beperkt tot de oude sRGB-wereld uit de jaren negentig.

Bouw mee met moderne webstandaarden en laat kleuren echt tot leven komen.
