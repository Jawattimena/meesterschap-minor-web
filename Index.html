Meer proberen te experimenteren

Waarom?

Ik merk dat ik al snel dingen ga maken of ideeën bedenk binnen technieken en manieren van werken die ik al ken. Daarbij kies ik vaak sneller voor kwaliteit en zekerheid dan voor experimenteren. Juist nu ik nog op school zit, voelt dit als het moment om die experimenten wel aan te gaan. Ik heb nu de ruimte om nieuwe dingen uit te proberen, fouten te maken en daarvan te leren.

Hoe?

Tijdens de meesterproef wil ik mezelf uitdagen om technieken, tools en manieren van werken op te pakken die ik normaal minder snel zou kiezen binnen het codeervak. Ik wil bewuster uit mijn comfortzone stappen en meer ruimte geven aan onderzoek en experiment, ook als het resultaat daardoor minder voorspelbaar is. Op die manier hoop ik niet alleen technisch meer te leren, maar ook creatiever en flexibeler te worden in mijn manier van werken.

JavaScript interacties

Waarom?

Ik merk dat ik JavaScript vaak liever uit de weg ga, omdat ik CSS simpelweg makkelijker en vertrouwder vind. Daardoor kies ik sneller voor oplossingen die vooral visueel sterk zijn, in plaats van interactief. Ik weet ook dat het bij JavaScript soms langer duurt voordat ik iets werkends heb om te laten zien, waardoor ik sneller teruggrijp naar wat ik al kan. Dit leerdoel sluit daarom ook aan op leerdoel 1, omdat ik mezelf wil uitdagen om meer te experimenteren en buiten mijn comfortzone te werken.

Hoe?

Tijdens de meesterproef wil ik bewuster interactieve onderdelen bouwen met JavaScript, ook als dit meer tijd kost of lastiger voelt. Ik wil mezelf uitdagen om niet direct voor de makkelijkste oplossing in CSS te kiezen, maar juist te onderzoeken wat JavaScript kan toevoegen aan een ervaring. Denk aan animaties, dynamische content of interacties die reageren op de gebruiker. Door dit vaker toe te passen hoop ik meer vertrouwen te krijgen in JavaScript en er comfortabeler mee te worden.

Begrijpen hoe technische samenwerkingen verlopen

Waarom?

Ik wil beter leren hoe technisch samenwerken binnen een team werkt, zodat ik hier later ook comfortabel mee om kan gaan. Tijdens de hackathon hebben we bijvoorbeeld gewerkt met het mergen van GitHub-branches, maar daar begreep ik toen eigenlijk nog niet zoveel van. Omdat samenwerken in teams later een belangrijk onderdeel wordt binnen het werkveld, lijkt het me waardevol om me hier meer in te verdiepen. Ik wil beter begrijpen hoe je samen aan één project werkt zonder elkaar in de weg te zitten, en hoe tools zoals GitHub daarin gebruikt worden.

Hoe?

Ik wil hier tijdens de meesterproef actief mee oefenen door samen te werken in GitHub en bewust merge conflicts op te lossen in plaats van ze uit de weg te gaan. Ook wil ik workshops volgen die over technisch samenwerken gaan, en vragen blijven stellen aan docenten en medestudenten wanneer ik ergens vastloop. Door hier vaker mee bezig te zijn hoop ik meer inzicht en vertrouwen te krijgen in technische samenwerking binnen een team.

Project

CSS

Dit project draait om het ontdekken van wat er allemaal mogelijk is met moderne CSS en HTML. Tijdens deze vierweekse cursus hebben we een interactieve interface gemaakt met alleen vanilla HTML en CSS, zonder JavaScript of frameworks te gebruiken. Het doel was om nieuwe CSS-technieken te leren en te onderzoeken hoe ver je kunt komen met alleen HTML en CSS.


Tijdens het project werkte ik volgens een “selector first” aanpak. Dat betekent dat we bijna geen classes en id’s mochten gebruiken, zodat we leerden werken met verschillende CSS-selectors en pseudo-classes zoals :has() en :target. Hierdoor heb ik geleerd hoe je interactie kunt maken zonder JavaScript.


Daarnaast heb ik gewerkt met technieken zoals popovers, anchor positioning, scroll snap en geneste CSS. Het project ging vooral over experimenteren, het schrijven van nette en overzichtelijke code en het bouwen van een interactieve gebruikerservaring met veel aandacht voor details.

Project

Hackathon

Dit project was een hackathon waarin we in één week tijd met een team een werkend prototype moesten ontwerpen en bouwen voor een echte opdrachtgever: SRON. SRON is het Nederlands instituut voor ruimteonderzoek en houdt zich bezig met wetenschappelijk onderzoek naar de ruimte, bijvoorbeeld met satellieten en instrumenten die gegevens verzamelen over planeten, sterren en de atmosfeer van de aarde.


Omdat we maar een week de tijd hadden, lag de focus sterk op samenwerken, snel keuzes maken en efficiënt werken. Tijdens dit project heb ik veel geleerd over werken in teamverband en het samenwerken binnen GitHub. Ik heb geleerd hoe je met branches werkt, code merged en samen aan dezelfde codebase kunt werken zonder elkaars werk te overschrijven. Dit was een belangrijk onderdeel van het project, omdat meerdere mensen tegelijk aan verschillende onderdelen van de website werkten.


Daarnaast werkten we volgens het principe “native web first”. Dat betekende dat we vooral HTML, CSS en waar nodig JavaScript gebruikten, zonder frameworks. Het doel was om een creatieve, interactieve en goed werkende website te bouwen met aandacht voor responsive design, toegankelijkheid en nette code. Naast het eindproduct was ook het proces belangrijk: samenwerken, communiceren, feedback verwerken en professioneel werken onder tijdsdruk stonden centraal tijdens de hackathon.

Project

Human Centered Design

Dit project draaide om het ontwerpen van een toegankelijke webapplicatie voor Roger, een slechtziende gebruiker die tijdens het lezen van digitale boeken annotaties wil kunnen maken. Gedurende vier weken heb ik meerdere prototypes getest en verbeterd op basis van feedback van Roger zelf. De focus lag op toegankelijkheid, screenreaders en het begrijpen van hoe blinde en slechtziende gebruikers websites ervaren en bedienen.


Tijdens het project heb ik veel geleerd over ARIA-labels, toetsenbordnavigatie, screenreaders zoals NVDA en VoiceOver en het belang van semantische HTML. Ook ontdekte ik hoe belangrijk het is om HTML-elementen alleen te gebruiken waarvoor ze bedoeld zijn. Door regelmatig met Roger te testen kreeg ik steeds beter inzicht in zijn behoeften en hoe kleine aanpassingen een groot verschil kunnen maken voor de gebruikservaring.

Project

API

Dit project draait om het bouwen van een webapplicatie met behulp van Astro, waarbij je werkt met verschillende web- en content API’s om een rijke en interactieve gebruikerservaring te maken. Het doel is om te ontdekken hoe krachtig het web is en hoe je moderne technieken kunt gebruiken om een app-achtige website te bouwen.


Tijdens het project ontwikkel ik een website die dagelijks een persoonlijke quote toont op basis van vragen en gebruikersinput. Hiervoor gebruik ik onder andere localStorage om gegevens zoals de naam en opgeslagen quotes te bewaren, een content API voor de quotes zelf, en een web/prompt API om de juiste quote te kiezen en de ervaring persoonlijker te maken.


Naast de technische kant leer ik ook hoe belangrijk het is om goed te plannen en structuur aan te brengen in je code en logica. Het project helpt mij beter begrijpen hoe API’s samenwerken, hoe je data opslaat en ophaalt in de browser, en hoe je een dynamische webapplicatie opbouwt met Astro.
