Conference: The Web You Want

17-04-2026 & 10-06-2026

The Web You Want

Twee dagen lang draaide het programma om de toekomst van het web. Van Europese digitale onafhankelijkheid en toegankelijkheid tot HTML-first ontwikkelen en de vraag hoe we het web persoonlijker, duurzamer en creatiever kunnen maken.

Dag 1

De eerste dag stond grotendeels in het teken van de afhankelijkheid van Europa van grote Amerikaanse technologiebedrijven. Verschillende sprekers bekeken dit vanuit hun eigen expertise, maar kwamen tot dezelfde conclusie: een groot deel van onze digitale infrastructuur draait op een klein aantal Amerikaanse platforms.

Dat is jarenlang praktisch geweest, maar brengt ook risico's met zich mee. Veranderende wetgeving, geopolitieke spanningen of beleidskeuzes van bedrijven kunnen directe gevolgen hebben voor Europese organisaties.

  • Afhankelijkheid bij geopolitieke spanningen
  • Vraagstukken rondom privacy en dataopslag
  • Innovatie en opbrengsten die vooral buiten Europa terechtkomen

Mogelijke oplossingen die werden besproken waren investeren in Europese cloud- en hostingpartijen, het verminderen van leveranciersafhankelijkheid en bewuster kiezen voor open standaarden en open source software.

Een klok tekenen met een laser

Tussen de serieuze discussies door was er ook ruimte voor experiment. Een spreker demonstreerde een zelfgebouwde laserprojector waarmee hij klokken kon "tekenen".

Door een grafiek te genereren en deze met een laser te projecteren ontstond een beeld dat als een wijzerplaat gelezen kon worden. Een mooi voorbeeld van technische creativiteit midden in een conferentie over digitale onafhankelijkheid.

Dag 2

Waar dag 1 draaide om infrastructuur en macht, voelde dag 2 veel persoonlijker. Toegankelijkheid, gebruiksvriendelijkheid en de essentie van het web stonden centraal.

Leonie Watson — leven met een onvolmaakt web

Leonie Watson, die blind is, gaf een van de meest indrukwekkende presentaties van het evenement. Ze wees op de ironie dat de European Accessibility Act, een wet die toegankelijkheid moet bevorderen, zelf slecht toegankelijk is.

  • Geen duidelijke kopstructuur
  • Geen navigeerbare lijsten
  • Veel complexe tabellen

Omdat de informatie moeilijk toegankelijk is via een schermlezer, gebruikt ze regelmatig AI-tools zoals ChatGPT om documenten samen te vatten en begrijpelijk te maken.

Ook in andere situaties helpt AI haar om barrières te overbruggen. Tijdens het online winkelen zijn productbeschrijvingen vaak onvoldoende. In de fysieke wereld gebruikt ze Meta-glasses om objecten te herkennen en haar omgeving te laten beschrijven.

AI werd niet gepresenteerd als gimmick, maar als een tolk tussen haar en een wereld die niet altijd voor haar ontworpen is.

Haar boodschap was duidelijk: bouw toegankelijkheid vanaf het begin in zodat toekomstige gebruikers niet telkens dezelfde obstakels hoeven te overwinnen.

Cyd Stumpel — terug naar HTML-first

Cyd Stumpel hield een pleidooi voor progressive enhancement en een terugkeer naar HTML als fundament van moderne webontwikkeling.

Haar bekende metafoor was die van een roltrap:

  • Een trap moet altijd bruikbaar zijn
  • Een roltrap is een verbetering bovenop die basis
  • Extra functionaliteit mag nooit de kern blokkeren

Daarmee contrasteerde ze progressive enhancement met graceful degradation. Haar voorkeur lag duidelijk bij eerst de basis bouwen en daarna pas verrijken.

Native accordions met <details>

Als voorbeeld gebruikte ze het native <details>-element. Met moderne CSS kun je dit tegenwoordig animeren zonder JavaScript. 

:root {
  interpolate-size: allow-keywords;
}

@supports (interpolate-size: allow-keywords) {
  @media (prefers-reduced-motion: no-preference) {
    ::details-content {
      height: 0;
      overflow: clip;
      transition: height 0.4s ease-out,
                  content-visibility 0.4s allow-discrete;
    }

    details[open]::details-content {
      height: auto;
    }
  }
}

Hiermee ontstaat een volledig native accordion die toegankelijk blijft en geen JavaScript nodig heeft.

De Popover API

Een tweede voorbeeld was de nieuwe Popover API waarmee interactieve navigatiemenu's gebouwd kunnen worden zonder JavaScript. 

<header>
  <button class="nav-toggle"
          popovertarget="menu">
    <span class="open-text">Menu</span>
    <span class="close-text">Close</span>
  </button>

  <nav id="menu" popover>
    <ul>
      <li><a href="/">Home</a></li>
    </ul>
  </nav>
</header>

Met :popover-open, @starting-style en clip-path kunnen vloeiende animaties worden toegevoegd. Voor oudere browsers kan via @supports selector(:popover-open) netjes worden teruggevallen op een eenvoudigere versie.

Volgens Cyd laat dit zien hoe krachtig moderne browsers inmiddels zijn: HTML en CSS vormen vaak een robuuster startpunt dan direct naar JavaScript grijpen.

Sara — whimsica11y en plezier op het web

Sara sloot af met een presentatie die ze "whimsica11y" noemde. Een combinatie van toegankelijkheid, experiment en plezier.

Ze liet zien dat klassieke webcultuur nog steeds bestaat:

  • Webrings
  • Blogrolls
  • Gastenboeken
  • Persoonlijke homepages

Websites zoals localghost, henry.codes en Neocities vormden voorbeelden van een web dat nog steeds persoonlijk en eigenzinnig is.

Daarbij stelde ze twee interessante vragen:

  • Moet alle kunst op het web altijd volledig toegankelijk zijn?
  • Wat is er gebeurd met het experimentele multimediale web?

Haar doel was niet om toegankelijkheid af te wijzen, maar om ruimte te behouden voor creativiteit en experiment. Een web dat niet alleen geoptimaliseerd is voor schaal, maar ook voor persoonlijkheid.

Conclusie

  • Europa zoekt naar meer digitale onafhankelijkheid
  • Toegankelijkheid blijft een fundamentele verantwoordelijkheid
  • HTML-first en progressive enhancement winnen terrein
  • Moderne browsers bieden steeds meer native mogelijkheden
  • Het web mag ook persoonlijk, creatief en speels blijven
Bouw een web dat toegankelijk, duurzaam én menselijk blijft.
