Minder JavaScript,
meer CSS

Als webontwikkelaars grijpen we vaak automatisch naar JavaScript. Een animatie? JavaScript. Een accordion? JavaScript. Een popup? JavaScript. Maar volgens Killian Valkhof doen we onszelf — en onze gebruikers — daarmee vaak tekort. Het web kan namelijk veel meer uit zichzelf dan we denken.

The Rule of Least Power

Een belangrijk principe dat Killian aanhaalt is The Rule of Least Power. Deze regel stelt dat je altijd de minst krachtige technologie moet gebruiken die je probleem kan oplossen.

HTML → CSS → JavaScript
  • HTML is het minst krachtig, maar ook het meest robuust.
  • CSS voegt presentatie en interactie toe.
  • JavaScript is het krachtigst — en daarmee ook het zwaarst.

Hoe krachtiger de technologie, hoe meer werk de browser moet doen: JavaScript moet worden gedownload, geparsed, uitgevoerd en onderhouden. Als iets met HTML of CSS kan, moet je daar beginnen.

Het web is backwards compatible

Een van de meest unieke eigenschappen van het web is dat het backwards compatible is. De allereerste website werkt vandaag de dag nog steeds.

  • Kennis van HTML raakt nooit echt verouderd
  • Oude technieken blijven functioneren
  • Nieuwe features bouwen voort op bestaande standaarden

Tegelijkertijd zijn er tegenwoordig betere, snellere en efficiëntere manieren om dingen te bouwen — vooral als je moderne HTML en CSS gebruikt.

JSX is gewoon HTML (met een omweg)

JSX voelt voor veel developers als iets nieuws, maar in essentie is het een dialect van HTML. Het beschrijft structuur — alleen via JavaScript. Dat maakt het makkelijk om:

  • semantiek te vergeten
  • native browsergedrag te overschrijven
  • oplossingen opnieuw te bouwen die de browser al biedt

Door eerst na te denken in HTML, bouw je automatisch toegankelijker en duurzamer.

Native oplossingen: accordions zonder JavaScript

Een klassiek voorbeeld is het accordion-menu. Vaak gebouwd met JavaScript, terwijl HTML hier al een perfect element voor heeft: 

<details>  
    <summary>Meer informatie</summary>
    <p>Dit is de verborgen content.</p>
</details>
  • Toegankelijk met toetsenbord en screenreaders
  • Geen JavaScript nodig
  • Werkt in alle moderne browsers

Het pijltje dat je ziet bij het openklappen heet de marker. Ook die kun je gewoon stylen of vervangen met CSS.

Appearance: none — volledige controle over styling

Form-elementen worden standaard gestyled door het besturingssysteem. Met:

appearance: none;

verwijder je die OS-styling en krijg je volledige controle.

  • display: none → het element verdwijnt volledig
  • appearance: none → het element blijft bestaan en werken, maar zonder standaard uiterlijk

Ideaal voor custom buttons, selects, checkboxes en radio buttons.

Animaties met respect voor de gebruiker

Niet iedereen kan goed tegen animaties. Daarom bestaat de media query:

@media (prefers-reduced-motion: no-preference) {
      /* animaties hier */
    }

Zo respecteer je gebruikersinstellingen zonder JavaScript. CSS kan niet alleen animeren, maar ook rekening houden met persoonlijke voorkeuren.

In-page transitions zonder JavaScript

Veel interacties binnen een pagina — hover-effecten, focus states, open- en dichtklappen — kunnen volledig met CSS. Dat is sneller, eenvoudiger en beter te onderhouden. JavaScript is hier vaak onnodig complex.

UX-details doen ertoe

Gebruik de handcursor alleen voor links.
De browser geeft hiermee een verwachting aan. Als je die verwachting breekt, wordt je interface verwarrender — hoe klein het detail ook lijkt.

Dialogs in plaats van alerts

In plaats van alert() en prompt() bestaat nu het native <dialog>-element:

<dialog open>
      <p>Weet je het zeker?</p>
      <button>OK</button>
    </dialog>
  • Beter te stylen
  • Toegankelijker
  • Onderdeel van HTML zelf

Bijblijven met het moderne web: Baseline

Het web ontwikkelt zich snel. Om bij te houden wat veilig te gebruiken is, bestaat Baseline.

Newly available

Ondersteund door alle core browsers. Interoperabel en bruikbaar.

Widely available

30 maanden later. Geschikt voor de meeste websites zonder support-zorgen.

Baseline helpt developers om moderne features te gebruiken zonder te gokken op browser support.

Conclusie

  • Het web is al heel capabel
  • HTML en CSS verdienen een grotere rol
  • JavaScript is waardevol, maar niet altijd nodig
  • Minder code betekent vaak betere performance, toegankelijkheid en onderhoud
Bouw met het web mee — niet tegen het web in.
